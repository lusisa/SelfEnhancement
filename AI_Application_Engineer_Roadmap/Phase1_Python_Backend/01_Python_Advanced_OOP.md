# Phase 1: Python 高阶与后端基础学习指南

## 1. 深入理解 Python 面向对象编程 (OOP)

面向对象编程是构建复杂系统的基石。请重点掌握：

*   **类的继承与多态**：如何在多个子类中复用代码并实现各自的特化。
*   **魔术方法 (Magic Methods)**：如 `__init__`, `__call__`, `__str__`, `__repr__`。这对于理解框架源码非常重要。
*   **属性描述符 (@property)**：如何在类中进行数据封装与校验。

**实践建议**：尝试用面向对象的思维写一个简单的员工管理系统或银行账户系统，支持账户开通、转账、流水打印等。

## 2. 深入理解高级特性

*   **装饰器 (Decorators)**：广泛用于 FastAPI 中的路由注册、权限校验等。要能手写带参数的装饰器。
*   **生成器 (Generators)**：使用 `yield`，在处理海量数据（如大型文档/日志分析）时非常有用。
*   **并发编程 (Concurrency)**：理解多线程 (threading)、多进程 (multiprocessing) 和异步协程 (asyncio)。
    *   **重点**：深入理解 `async/await`，因为后续的大模型调用（网络 I/O 密集型任务）和 FastAPI 框架都重度依赖异步编程。

**实践建议**：写一个爬虫，分别用单线程、多线程、异步三种方式抓取同一批网页，对比速度。

## 3. 设计模式 (Design Patterns)

设计模式是写出高可维护性代码的蓝图。不要死记硬背，重点理解解决什么问题。

*   **单例模式 (Singleton)**：常用于数据库连接池、日志记录器。
*   **工厂模式 (Factory)**：常用于根据不同输入创建不同的 LLM 实例（如 OpenAI实例, 百度千帆实例）。
*   **策略模式 (Strategy)**：常用于切换不同的业务算法（如不同的文本分块策略）。
*   **观察者模式 (Observer)**：常用于事件驱动架构。

**实践建议**：在之前的项目（如爬虫或管理系统）中重构代码，强行套用至少两种设计模式。

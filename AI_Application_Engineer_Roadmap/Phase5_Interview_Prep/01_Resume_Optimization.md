# Phase 5: 简历优化指南 (针对车企 AI 岗位)

由于您是非科班出身、且目标是 4 年经验以上的高级岗位，你的简历必须从“我懂这个技术”转变到“我能用这个技术解决真实的业务痛点”。

## 1. 核心包装策略：技术 + 行业/场景
JD 中极其强调“业务场景深度集成”。在编写简历时，请遵循 **STAR 法则 (Situation, Task, Action, Result)**。

**❌ 错误的写法 (像学生/初级开发)：**
"使用 Python 和 LangChain 开发了一个 RAG 问答系统，用到了 Milvus 向量数据库。"

**✅ 优秀的写法 (像资深架构师)：**
"**汽车底盘设计规范智能检索引擎**：
针对研发中心底盘团队查阅设计规范(国标/企标)效率低、易遗漏的痛点 (Situation/Task)。
设计并实施了基于 LlamaIndex 和 Milvus 的企业级 RAG 架构。采用父子文档层级索引策略并引入混合检索(BM25+Dense)，结合开源 BGE-Reranker 进行重排 (Action)。
系统上线后（或：测试验证显示），检索准确率提升至 92%，使工程师单次规范核对时间从平均 15 分钟缩短至 1 分钟以内 (Result)。"

## 2. 弱化劣势，强调可迁移能力

*   **没有大型研发机构经验怎么办？**
    挖掘你过去工作中的**跨部门沟通**、**需求分析**、**流程梳理**经验。
    在简历中增加一个模块：**【综合素质与业务分析能力】**，描述你如何将模糊的业务需求拆解为技术可落地的方案。对应 JD 里的：“具备极强的逻辑思维能力和跨部门沟通能力”。
*   **强调你的学习与演进能力**：
    JD 提到“持续跟踪全球 AI 前沿技术”。你可以在简历中放上你的 GitHub 链接、技术博客链接，或者注明经常阅读 arXiv 上的顶会论文。

## 3. 技能清单 (Skills) 书写建议
*   **精通**：Python, Prompt Engineering, 常见的 RAG 优化策略。
*   **熟练掌握**：FastAPI, LangChain, LlamaIndex, Chroma/Milvus, 多线程与并发编程。
*   **熟悉**：车企研发体系 (PLM, ERP), Agent 架构 (ReAct, Tool Calling), 数据结构与算法。
*   **了解**：PyTorch (能看懂基础代码即可，不强求会训练大模型), 微调 (SFT) 基础概念。

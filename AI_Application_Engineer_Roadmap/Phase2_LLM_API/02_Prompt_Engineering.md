# Phase 2: Prompt Engineering (提示词工程)

在 AI 应用开发中，Prompt 就是新的编程语言。写好结构化提示词是每个开发者的必修课。

## 1. 结构化提示词的基础架构

一个优秀的工业级提示词通常包含以下结构：

1.  **Role (角色设定)**: 让大模型带入特定的专家身份。
    *   "你是一位拥有 20 年经验的车企资深架构师，精通自动驾驶仿真系统..."
2.  **Context (背景上下文)**: 提供完成任务必须的信息。
    *   "我们目前正在使用 Teamcenter PLM 系统管理零部件 BOM 表，需要将其中存在的不规范数据清洗出来..."
3.  **Task (具体任务)**: 明确指出你要模型做什么。
    *   "请帮我从下面这段杂乱文本中抽取 '零部件编号'、'材料'、'公差' 三个字段..."
4.  **Constraints (限制条件/输出格式)**: 对输出做严格约束。
    *   "必须且仅输出合法的 JSON 格式。不要包含任何多余的解释，不要使用 Markdown 代码块包裹..."
    *   "如果找不到对应的字段，请填写 'null'..."
5.  **Examples (Few-Shot 示例)**: 极其重要！给模型打个样。

## 2. 进阶技巧 (面试加分项)

### Few-Shot Prompting (少样本提示)
仅通过指令往往不够准确，提供 1-3 个完整的输入输出样例，能大幅提升大模型的理解力和格式依从性。

### Chain of Thought (CoT, 思维链)
在要求模型给出最终答案之前，先让它"想一遍"。
*   经典咒语："Let's think step by step." (让我们一步步来思考)。
*   工业级用法：让模型先输出一个 `<thinking>` 或 `\"reasoning_steps\"` 字段，分析完后再给出 `"final_result"`。

## 3. 练习方式
不要只在网页上玩 ChatGPT！
使用 OpenAI Playground，调整 Temperature 为 0，输入各种不规则的长文本，尝试写出能完美提取特定字段且返回 JSON 格式的 Prompt。
